# 国内生涯与智慧教育平台竞品调研：围绕“人生实验室”的产品机会

调研日期：2026-07-14  
调研对象：科大讯飞智慧教育云平台/智慧教育方案、好未来学业规划相关能力、立思辰智慧教育、希沃智慧教育、云校生涯、VectorLab Youth-Pi  
调研目标：从产品介绍网站中提取核心信息，判断国内核心产品在“学生画像动态化—资源集中整合与多因素推荐匹配—行动转化与闭环反馈校准—画像再更新”链路上的覆盖情况，并为“人生实验室”产品理念提供竞品参照。

## 一、我们的产品理念与判断框架

本项目希望建立的核心理念是：

> 人生实验室：帮助学生把模糊兴趣变成可验证假设，通过一次低成本真实体验获得新证据，再修正下一步判断。

对应的核心业务流程是：

1. 学生画像动态化：把学业、兴趣、能力、五育表现、活动经历、反思记录、教师/导师反馈等转化为可追溯成长画像。
2. 资源集中整合 + 多因素推荐匹配：整合课程、社团、竞赛、实践、导师、访谈、项目、志愿服务等机会，根据画像、阶段、门槛、时间成本、风险与可获得证据进行推荐。
3. 行动转化 + 闭环反馈校准：把推荐变成一次低成本、可执行、可记录的真实体验任务。
4. 行动数据回流画像：学生完成行动后，通过反思、作品、评价、结果等新证据更新画像。
5. 往复循环：不断推动学生从“我好像感兴趣”走向“我用真实行动验证过”。

因此，本次竞品调研不是只问“平台有什么功能”，而是重点判断它们是否支持以下五件事：

| 判断维度 | 关键问题 |
|---|---|
| 画像 | 是否能形成动态学生画像，且不只依赖一次测评？ |
| 资源 | 是否有结构化资源库，覆盖课程、实践、升学、职业或成长机会？ |
| 推荐 | 是否能给出可解释、多因素匹配的推荐？ |
| 行动 | 是否能把建议转化为低成本真实体验任务？ |
| 反馈 | 是否能把行动结果回流，校准下一轮判断？ |

其中与学生成长/学业规划关系较近的能力包括：

- 课后服务、AI 学习机：补充个性化学习资源和学习陪伴；
- 区域/学校治理平台：偏管理端、数据端能力。

#### 对“人生实验室”流程的覆盖

| 环节 | 覆盖程度 | 判断 |
|---|---|---|
| 学生画像动态化 | 较强 | 讯飞在学业数据、五育评价、心理健康、学习诊断方面能力较强，适合形成学生多维画像。 |
| 资源集中整合 | 中等 | 资源主要围绕学科知识、练习、教学内容、课后服务和学校应用，职业/实践/社会体验资源不突出。 |
| 多因素推荐匹配 | 较强但偏学业 | 个性化学习手册和 AI 学习机强调诊断后推荐学习路径，推荐逻辑更偏“知识薄弱点—学习资源”。 |
| 行动转化 | 中等 | 能转化为学习任务、练习任务、课堂活动，但不明显支持“职业/兴趣假设验证式体验任务”。 |
| 反馈校准 | 较强但偏学业闭环 | 学习结果可以回流诊断，形成学业闭环；生涯探索行动的回流机制未见明显公开说明。 |

#### 竞品启示

科大讯飞的优势是 AI、大数据、学情诊断和教育治理基础设施。它解决的是“学生学习状态如何被识别、教学如何更精准”的问题。对我们而言，讯飞像是“学业诊断引擎”和“教育数据底座”的标杆。

但“人生实验室”不能只做学业诊断。我们的差异化应放在：

- 不只判断“你哪道题不会”，还判断“你对什么方向有假设”；
- 不只推荐学习资源，还推荐真实体验；
- 不只回收练习结果，还回收体验证据、反思和导师反馈；
- 不只服务教师精准教学，还帮助学生主动做人生小实验。

好未来的优势是学习内容、学习服务品牌、AI 学习陪伴和学生高频学习场景。它擅长解决“学生遇到学习问题后如何快速获得讲解和练习支持”。

对“人生实验室”来说，好未来提醒我们：

- 学生产品必须足够高频，否则画像和反馈很难动态更新；
- AI 不能只生成一次规划报告，而要在学生每天遇到问题时提供陪伴；
- 内容资源必须结构化，否则推荐无法解释。

但好未来公开能力仍偏“学科学习提升”。我们的差异化应强调“学习之外的成长证据”：

- 把一次访谈、一次社团体验、一次项目挑战也当作学习；
- 把学生反思和作品作为比题目正确率更宽的成长数据；
- 把 AI 从答题助手扩展为探索助手。

由于官网来源无法确认，本报告不对立思辰智慧教育做细颗粒功能判断。若后续能提供准确官网或产品页，可补充以下维度：

在后续路演或报告中，我们应避免说“某平台已经/没有某功能”而没有证据；更稳妥的表达是“公开资料未见充分说明”。

#### 竞品启示

希沃的优势是“离学生最近的学校硬件和课堂入口”。如果一个产品想在学校里真正落地，必须嵌入教师和班级已经在用的工作流，而不是另起一个孤岛平台。

对“人生实验室”来说，希沃带来的启示是：

- 我们不应只做一个学生 App，还要能嵌入班主任、学科教师、心理老师和家长的已有工作流；
- 体验任务最好能在班级、课堂、社团、综合实践活动里自然发生；
- 五育综评和课堂活动记录可以成为生涯画像的一部分；
- 但我们要补的是“学生自主提出假设、选择体验、反思校准”的链路，而不是再做一个课堂工具。

### 6. VectorLab Youth-Pi：创生教育成长与履历平台

官网入口：

- VectorLab / Youth-Pi 官网：https://www.vectorlab.com.cn/#modules

#### 官网公开能力

VectorLab 官网将 Youth-Pi 定位为“创生教育成长与履历平台”，页面标题为 Talent Youth Pi。官网介绍中，“创生教育”理念强调以学生激情与内驱为中心，以挑战性问题牵引学习，让学生在真实问题中完成从知识学习到问题探索、从项目实践到自我生长的转变。

Youth-Pi 的公开能力主要包括：

- 从真实挑战出发：围绕前沿科技、真实场景与社会议题设计学习任务，让学生带着问题进入项目；
- 课程模块支撑实施：沉淀课程资源、任务模板、学习材料与评价量规，帮助学校组织项目式学习；
- 项目过程证据沉淀：支持报告、代码、模型、实验记录、展示材料、反思日志等多元成果提交；
- 多元反馈推动迭代：教师、导师与评审基于评价标准提供反馈，帮助学生修正方案、优化作品；
- 过程评价看见成长：记录学生在调研、设计、实验、协作、表达、反思中的真实表现；
- 可信成长履历：将课程参与、作品证据、评价摘要、导师反馈与认证信息沉淀为可验证成长记录；
- 多角色协同：学校管理员配置组织与班级，教师实施教学与评价，学生沉淀作品证据，顾问与校管审核后生成履历快照。

#### 对“人生实验室”流程的覆盖

| 环节 | 覆盖程度 | 判断 |
|---|---|---|
| 学生画像动态化 | 较强 | Youth-Pi 通过项目过程、作品证据、评价摘要、导师反馈和认证信息形成成长履历，明显强于一次性测评画像。 |
| 资源集中整合 | 中等到较强 | 平台沉淀课程资源、任务模板、学习材料与评价量规，偏项目式学习与创新教育资源。 |
| 多因素推荐匹配 | 中等 | 官网更强调项目组织、过程记录和履历生成，未突出 AI 个性化推荐或多因素机会匹配。 |
| 行动转化 | 较强 | 以真实挑战、项目任务、成果提交为核心，天然把学习转化为行动和作品。 |
| 反馈校准 | 较强 | 教师、导师与评审的多元反馈，以及学生作品迭代，构成比较清晰的行动反馈闭环。 |

#### 竞品启示

Youth-Pi 是本次调研中最接近“成长证据驱动”理念的国内竞品之一。它不像传统升学规划产品那样以测评、院校库、专业库为中心，而是以“真实项目—作品证据—评价反馈—成长履历”为主线。

这对“人生实验室”有三个重要启发：

1. 成长证据要有实体承载。学生说“我喜欢 AI”并不足够，平台应该记录他是否参与过 AI 项目、是否提交过代码或模型、是否完成过实验记录、是否获得导师反馈。
2. 体验任务要尽量产生作品。比起只写一段感想，有作品、报告、访谈记录、实验结果或展示材料，才更容易成为可验证证据。
3. 履历不是终点，而应成为下一轮推荐的输入。Youth-Pi 强在履历沉淀；人生实验室可以进一步补上 AI 动态画像、资源匹配和下一次低成本实验推荐。

#### 与“人生实验室”的差异

| 维度 | Youth-Pi | 人生实验室 |
|---|---|---|
| 核心定位 | 创生教育成长与履历平台 | AI 生涯成长实验平台 |
| 切入口 | 项目式学习、创新教育、真实挑战 | 模糊兴趣假设、低成本体验、持续校准 |
| 核心资产 | 作品证据、项目过程、可信履历 | 动态画像、机会匹配、行动卡片、反馈校准 |
| 行动闭环 | 强，围绕项目实践和作品迭代 | 强，围绕“假设—体验—证据—修正”循环 |
| 推荐能力 | 官网未突出 | 应作为核心能力：推荐下一次可验证实验 |
| 适用边界 | 更偏创新教育、PBL、科创/探究项目 | 可扩展到职业访谈、社团体验、课程试听、志愿服务、项目挑战等更广成长场景 |

因此，Youth-Pi 可以被视为“成长履历/项目实践型”竞品。它证明国内市场已经出现从“测评—推荐”转向“实践—证据—履历”的产品方向；而我们要继续向前一步，把履历证据变成动态画像和下一轮行动推荐的燃料。

## 三、横向对比：谁更接近“人生实验室”？

| 平台 | 主要强项 | 更像解决什么问题 | 与“人生实验室”的距离 |
|---|---|---|---|
| 科大讯飞 | AI、大数据、学情诊断、精准教学、五育评价、心理健康 | 如何识别学生学习状态，并提升教学精准度 | 有画像与反馈基础，但偏学业和学校治理，缺真实体验任务 |
| 好未来 | 学习内容、AI 学习工具、学习服务、学生高频学习场景 | 学生遇到学习问题后如何获得讲解、练习和陪伴 | 有高频学习入口，但偏学科提分，缺生涯实践资源 |
| 立思辰智慧教育 | 本次无法从给定官网验证 | 暂不判断 | 官网链接不匹配，无法实证 |
| 希沃智慧教育 | 课堂入口、教学硬件、教师工具、五育综评、AI 教学空间 | 学校如何数字化教学、评价和管理 | 能沉淀校内行为数据，但不以学生生涯探索为中心 |
| 云校生涯 | 本次无法从官网验证 | 暂不判断 | 官网不可解析，无法实证 |
| VectorLab Youth-Pi | 项目式学习、作品证据、过程评价、成长履历 | 学生如何通过真实项目证明自己的创新能力与成长过程 | 与“人生实验室”高度相关，强在行动证据和履历沉淀，弱在 AI 推荐下一次实验 |

总体来看，国内主流智慧教育产品通常强在三类能力：

1. 学业诊断与精准教学；
2. 教师/学校数字化管理；
3. 综合素质评价与学习资源供给。

Youth-Pi 进一步补上了“项目实践—作品证据—成长履历”这一类能力，但多数平台仍没有完整把“兴趣—假设—低成本体验—证据—校准”作为产品主线。换句话说，它们更多在回答：

- 学生哪里学得不好？
- 老师如何教得更精准？
- 学校如何管理得更高效？
- 学生综合素质如何被记录？

而“人生实验室”要回答的是：

- 学生如何把模糊兴趣变成一个可验证的小假设？
- 如何找到一次低成本真实体验？
- 如何判断这次体验给了我什么新证据？
- 我下一步该坚持、调整还是放弃？

这就是我们的差异化空间。

## 四、竞品对我们业务流程的启示

### 1. 学生画像动态化：不要只做测评画像，要做证据画像

科大讯飞、希沃都具备学业、五育、心理或课堂数据入口。它们说明学生画像不是难点，难点是画像是否能被学生理解、被持续更新、被用于下一步行动。

我们的画像应避免三种问题：

- 一次性测评后固定标签；
- 画像只给老师和家长看，学生自己看不懂；
- 画像只服务管理，不服务学生决策。

建议采用“证据画像”：

| 画像维度 | 证据来源 |
|---|---|
| 兴趣假设 | 学生自述、浏览收藏、活动报名、反思文本 |
| 能力倾向 | 学业表现、作品、项目结果、教师评价 |
| 行动风格 | 任务完成率、主动求助、协作记录、复盘质量 |
| 价值偏好 | 访谈反思、职业体验记录、导师反馈 |
| 风险支持 | 心理状态、压力信号、家校反馈，需严格权限保护 |

### 2. 资源集中整合：从“资源库”升级为“机会实验库”

好未来强在学习内容资源，希沃强在课堂资源，科大讯飞强在学情和教学资源。但“人生实验室”不能只是课程、题库和讲座集合，而应形成“机会实验库”。

每个资源都应被标注为一个可体验机会：

| 字段 | 示例 |
|---|---|
| 机会类型 | 职业访谈、社团体验、竞赛入门、项目挑战、课程试听、志愿服务 |
| 适合年级 | 初一/初二/初三/高一/高二/高三 |
| 前置条件 | 无基础/需数学基础/需英语表达/需作品准备 |
| 时间成本 | 30分钟、1小时、1周、1个月 |
| 可获得证据 | 访谈记录、作品、活动证明、导师评价、学生反思 |
| 风险提示 | 时间冲突、难度偏高、需要家长同意、可能产生费用 |
| 推荐理由 | 与学生兴趣假设、能力证据、阶段目标的对应关系 |

### 3. 多因素推荐匹配：推荐的不是答案，而是下一次实验

科大讯飞和好未来的推荐逻辑更偏学习路径：诊断薄弱点，然后推荐资源。我们的推荐逻辑要多一层：诊断学生的兴趣假设，然后推荐一次可以验证的行动。

推荐不应直接输出“你适合当医生/工程师/设计师”，而应输出：

- 你的当前假设是什么；
- 这个假设依据哪些证据；
- 还有哪些证据不足；
- 推荐你做哪一个低成本实验；
- 做完后如何判断假设更可信还是需要修正。

示例：

> 你最近对“心理学”感兴趣，但目前证据主要来自短视频和自我感受，缺少真实接触。建议本周完成一次“校园心理老师访谈 + 30分钟心理科普课程体验”，提交 3 个问题、1 段反思和 1 个下一步判断。完成后再决定是否进入“心理学入门书单/志愿服务/生涯访谈”下一轮实验。

### 4. 行动转化：核心不是“规划”，而是“最小可验证行动”

国内智慧教育平台普遍擅长做“测、评、教、学、管”，但相对弱在学生自驱行动。我们的产品应把行动卡片作为核心对象。

一张行动卡片至少包含：

- 实验假设：我想验证什么；
- 推荐原因：为什么推荐我做这个；
- 行动步骤：我具体做什么；
- 时间成本：多久完成；
- 提交证据：完成后上传什么；
- 反思问题：我学到了什么、还想不想继续；
- 反馈对象：老师、导师、家长或 AI 如何反馈；
- 下一步分叉：继续深入、换方向、补能力、延后探索。

这比“测评报告 + 专业库”更贴近中学生真实成长。

### 5. 闭环反馈校准：把失败也当作有价值证据

学业平台通常把结果分成对/错、会/不会、完成/未完成。但人生探索不能这样简单判断。

一次体验后，学生可能发现：

- 我没那么喜欢这个方向；
- 我喜欢理念，但不喜欢日常工作方式；
- 我能力暂时不够，但愿意补；
- 我原以为不喜欢，体验后发现有兴趣；
- 我喜欢这个方向，但不想走传统升学路径。

这些都是高价值证据。产品应把“兴趣下降”“放弃方向”“暂缓探索”也视为有效结果，而不是失败。

## 五、我们相对国内竞品的产品定位

可以把国内竞品大致分成三类：

| 类型 | 代表 | 核心能力 | 局限 |
|---|---|---|---|
| 学业诊断型 | 科大讯飞、好未来 | 学情识别、学习资源推荐、AI 讲解、精准教学 | 偏学科提分，难覆盖真实世界探索 |
| 学校数字化型 | 希沃、科大讯飞 | 课堂工具、教师工具、五育评价、治理平台 | 偏管理和教学效率，学生主体性不足 |
| 生涯咨询型 | 生涯类平台 | 测评、职业库、专业库、升学规划 | 容易停留在报告和路径建议，行动闭环不足 |
| 成长履历/项目实践型 | VectorLab Youth-Pi | 真实项目、作品证据、过程评价、可信履历 | 强在实践证据，推荐匹配和生涯方向校准仍可拓展 |

“人生实验室”的定位应是第四类：

> 成长实验型平台：以学生真实行动为中心，把测评、资源、推荐和反馈组织成持续迭代的成长实验闭环。

它的核心不是“告诉学生答案”，而是“帮学生设计下一次验证”。

## 六、建议写入产品方案的差异化表达

### 1. 产品主张

人生实验室不是传统生涯测评平台，而是一个帮助中学生持续验证未来可能性的 AI 成长实验系统。它把学生的模糊兴趣转化为可验证假设，把校内外资源转化为低成本体验任务，把每次行动结果沉淀为新证据，并持续校准学生画像和下一步推荐。

### 2. 关键差异

| 传统平台 | 人生实验室 |
|---|---|
| 一次测评生成报告 | 多源证据持续更新画像 |
| 推荐职业/专业答案 | 推荐下一次低成本实验 |
| 资源库静态浏览 | 机会实验库结构化匹配 |
| 学生被动接收建议 | 学生主动选择、行动、反思 |
| 结果偏升学/提分 | 结果包括兴趣变化、能力证据、行动风格 |
| 反馈停留在分数或评价 | 反馈回流画像并生成下一轮实验 |

### 3. MVP 功能建议

第一阶段不必做“大而全生涯平台”，建议优先做 5 个核心模块：

1. 兴趣假设生成器：把学生的模糊表达转成可验证假设。
2. 机会实验库：整合校内课程、社团、竞赛、访谈、实践和导师资源。
3. AI 行动卡片：把推荐拆成一次 30 分钟到 7 天内可完成的低成本行动。
4. 成长证据档案：记录学生提交的反思、作品、评价和结果。
5. 反馈校准引擎：基于行动结果更新画像，并给出下一轮分叉建议。

## 七、结论

从官网公开信息看，科大讯飞、好未来、希沃等国内核心智慧教育平台已经在学业诊断、AI 学习、精准教学、五育评价、课堂工具和学校数字化方面形成较强能力。这些平台能帮助学校更好地看见学生、帮助教师更精准地教学、帮助学生更高效地学习。VectorLab Youth-Pi 则提供了另一类重要参照：通过真实项目、作品证据、过程评价和可信履历，让学生成长不只停留在测评结果或课堂表现中。

但围绕“中学生生涯成长”这一主题，多数智慧教育平台共同的不足是：更关注学习与管理闭环，较少以学生的真实探索行动为中心；更擅长推荐学习内容，较少推荐真实世界体验；更重视诊断结果，较少把“行动后的新证据”作为下一轮推荐的核心依据。Youth-Pi 已经向“行动证据”迈出一步，但它更偏项目履历平台；人生实验室还应把这些证据进一步用于动态画像、机会匹配和下一次低成本实验推荐。

因此，“人生实验室”的机会非常清楚：

> 国内竞品大多在帮学生“学得更好”，而我们要帮学生“用一次次低成本真实体验，逐步知道自己想成为什么、适合怎样成长、下一步该怎么试”。

如果能把学生画像、资源匹配、行动卡片、反馈校准做成循环，人生实验室就能从传统生涯规划的“给答案”，升级为面向中学生成长的“做实验”。
